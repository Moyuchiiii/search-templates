# Search Templates

AIエージェントによる調査・リサーチのプロンプトテンプレート集。

## 基本方針

- 日本語で応答する
- 調査結果は省略・要約せず全文記録する
- すべての情報にソースURL（出典元リンク）を付記する
- レポートはHTML単一ファイルで出力する

## 調査の実行方法

IMPORTANT: ユーザーが「○○を調査して」「○○のリサーチ」等の指示を出したら、以下のフローを自動で実行すること。

### 実行フロー

1. `_template/00-orchestrator.md` を読んで全体フローを把握する
2. ユーザーの指示内容に合った `[ドメイン名]/prompt.md` を探す
   - **存在する**: そのまま読み込んで Phase 1 へ
   - **存在しない**: `_template/00a-generate-prompt.md` に従い prompt.md を自動生成し、ユーザーの確認を取ってから Phase 1 へ
3. `prompt.md` の `## 事前読み込み` に記載されたファイルを読む
4. Phase 1〜8 を順次実行する（各フェーズの詳細は `_template/01-08`）

prompt.md がない新規テーマでも、テーマ名とドメイン名を伝えるだけで調査を開始できる。
手動で prompt.md を作りたい場合は `_template/example-prompt.md` を参考にする。

### 出力先

- 生データ: `[ドメイン名]/YYYY-MM-DD/調査結果_YYYY-MM-DD.md`
- レポート: `[ドメイン名]/YYYY-MM-DD/レポート_YYYY-MM-DD.html`
- トラックレコード: `[ドメイン名]/track_record.md`

## テンプレート構成

| フェーズ | ファイル | 内容 |
|---------|---------|------|
| 0 | `00-orchestrator.md` | 全体フロー制御・並列実行ルール |
| 0.5 | `00a-generate-prompt.md` | prompt.md 自動生成（新規テーマ用） |
| 1 | `01-clarify.md` | 要件明確化・サブ質問分解 |
| 2 | `02-persona.md` | ペルソナ駆動調査（STORM方式） |
| 3 | `03-search-strategy.md` | 検索戦略・多言語・鮮度・MCP連携 |
| 4 | `04-data-collection.md` | データ収集・記録ルール |
| 5 | `05-verification.md` | 交差検証・コンセンサス判定 |
| 6 | `06-analysis.md` | 分析・スコアリング・市場分析 |
| 7 | `07-report.md` | レポート生成（HTML・テーマ・品質ルール） |
| 8 | `08-track-record.md` | 予測トラックレコード |

## モデル選択

| 用途 | モデル |
|------|--------|
| 全体制御・判断 | Opus |
| 調査・実行（デフォルト） | Sonnet |
| 定型転記・フォーマット変換 | Haiku |

## エージェント構成

```
リード（Opus）
  └── 調査エージェント（Sonnet）× 並列
        └── [大規模時] サブエージェント（Sonnet/Haiku）
```

- 独立したカテゴリ/国は並列エージェントで実行する
- 各エージェントへの指示は自己完結にする（曖昧な委譲は禁止）
- エージェント定義: `.claude/agents/researcher.md`

## 品質ルール

- 検索クエリは使ったもの全件を記録する（省略禁止）
- 収集した情報は全文記載する（「...」「以下省略」は禁止）
- ソースが確認できない情報は「⚠️ ソースなし」として分離する
- レポートに含める事実情報には例外なくソースURLを付記する
- 「たぶん通る」「直ったはず」等の未検証の完了宣言は禁止

## 新しい調査の追加方法

### 方法A: 自動生成（推奨）

テーマとドメイン名を伝えるだけでOK。Phase 0.5 が prompt.md を自動生成する。

例: 「Webマーケティングを `web-marketing` というドメイン名で調査して」

### 方法B: 手動作成

1. `_template/example-prompt.md` をコピーして `[調査名]/prompt.md` に配置する
2. 各セクションを記述する
3. 実行する

必須セクション: 基本情報 / 事前読み込み / ペルソナ設定 / 調査カテゴリ / 検索クエリ / レポートセクション

## コミット規約

- メッセージは日本語で記述
- 形式: `<種別>: <概要>`
- 種別: feat / fix / docs / chore
