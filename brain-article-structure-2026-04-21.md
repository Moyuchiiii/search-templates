## エグゼクティブサマリー

1. **目次は必須インフラ**: 売れている商品は全て商品ページの冒頭に目次が表示されている。「目次はありません」の商品でも本文内に番号付き見出しで構造を示している。目次なしは致命的な機会損失。
2. **冒頭の型は「ペイン提示→実績提示→ベネフィット3点→目次」の4点セット**: どの商品も冒頭500文字以内でペインポイントを出し、著者の実績（数字付き）を見せ、購入後の未来（3項目前後）を提示してから目次に入る。
3. **無料ゾーンは「第1章の途中まで」が主流**: 教科書型でも実録型でも、無料ゾーンの末尾は「ここから先でやること一覧（10〜20項目）を箇条書きで煽ってから」切れる。いわゆる「見せ切り型フック」が標準。

### 見出しタイトルの型

**具体性の高い型（売れてる商品に多い）**:
- 「なぜ日本語LPは9割崩れるのか」（Why型）
- 「AIを使ってる？AIに使われている？」（問いかけ型）
- 「僕の前から"不可能"という文字が消えた日」（ドラマ型）
- 「9割が挫折する本当の理由」（逆張り型）
- 「【証拠】これが"創造神"の力」（証拠提示型）

**抽象的でダメな型（避けるべき）**:
- 「第1章: 基礎」
- 「2. ツールの使い方」
- 「3. 実践」
- 「まとめ」

### 部構成の有無

調査した8商品中、「第○部」という部構成を採用している商品はなかった。代わりに:
- **Day別カリキュラム**（リツト・しゅうへい）
- **章番号 + 意味ある章タイトル**（まな・こうめい）
- **番号 + 特典番号のハイブリッド**（トシノ）
が主流。

### 各章への付加情報

調査した範囲では章ごとの「所要時間」「ページ数」の明記はなかった。ただし:
- しゅうへいは「動画○分」という動画尺を各章見出しに記載
- ユニコは「30種類のスタイルテンプレート」と量を数値で示す
- トシノは各特典に文字数（「約5,000文字のマニュアル」）を記載

## 5. 本文途中の仕掛け

### 販売部数の更新ログ配置

しゅうへいの商品に特徴的なパターン。本文の最冒頭に更新ログを置く:
```
[UPDATE]
3/27：発売初日2時間で200部突破
3/29：発売3日で600部突破
4/3：発売1週間で800部突破
（追加コンテンツの更新ログ）
```
これにより「売れている事実」「コンテンツが成長している事実」を同時に示す。

### Before/After スクショの配置

- リツト: 「月間約80時間（3.3日分）節約」をテキストで示し、スクショを1-3節に配置
- ユニコ: 横動画・縦動画のサンプル動画リンクを冒頭1-2節に配置（「このような動画も作れます」）
- メラメラくん・まな: 実績スクショを著者紹介節に配置（確認できた範囲）

### 購入者の声セクションの配置

| 商品 | レビューの配置位置 |
|------|------------------|
| しゅうへい | 目次の直後（第1節として「1. 購入者の口コミ」） |
| リツト | 目次確認できず（ページ下部のレビュータブは全商品共通） |
| トシノ | サイドバー目次に「購入者のレビュー」節なし（ページ下部のみ） |
| まな | 本文中に「実績者多数輩出」として文章に組み込み |
| ユニコ | ページ下部のレビュータブのみ（4.9★ 117件） |

## 7. 売れる構成 必須チェックリスト（15項目）

brain-runスキルに組み込む用のチェック項目（重要度順）:

**【目次・構造】**
- [ ] 1. 冒頭に番号付き目次がある（サイドバー目次 + 本文内目次の両方）
- [ ] 2. 章タイトルは「第1章: 基礎」ではなく「なぜ○○は9割失敗するのか」「Day1: ○○で最初の感動体験」等、具体的な内容を示している
- [ ] 3. 章数は6〜18章（少なすぎると「薄い」、多すぎると読めない）
- [ ] 4. 教科書型ならDay別 or Chapter別のカリキュラム表がある

**【冒頭フック・ペイン提示】**
- [ ] 5. 冒頭300文字以内でターゲット読者のペインポイントを3〜6項目列挙している
- [ ] 6. 著者の実績数字（部数・売上・期間・フォロワー等）が冒頭に出ている
- [ ] 7. この記事で得られること（ベネフィット）を3〜8項目で提示している

**【無料ゾーンの設計】**
- [ ] 8. 無料ゾーン末尾に「ここから先でやること一覧」または「第1章の冒頭1文」がある
- [ ] 9. 有料ゾーン直前に価格の正当化フレーズがある（「動画1本あたり○円」「開発に○時間かけた」等）
- [ ] 10. 購入ボタンのすぐ上に具体的な行動CTA（「今すぐボタンを押して○○を受け取ってください」）がある

**【社会的証明】**
- [ ] 11. 購入者の声が本文内（目次や冒頭節として）組み込まれている（ページ下部だけではダメ）
- [ ] 12. 販売実績（部数・売上）が本文内に明示されている
- [ ] 13. 発売後は[UPDATE]ログ（「○日で○部突破」等）を冒頭に追記している

**【コンテンツ価値の見せ方】**
- [ ] 14. 同梱物（テンプレート・動画・特典）の一覧が具体的な数量・内容付きで示されている
- [ ] 15. 「やらないこと」「おすすめしない人」リストがある（スコープ明示と属性選別）

現記事ファイル: `D:/Claude/content-pipeline/brain/today/brain_20260421_claude_design_nihongo_lp.html`

確認した冒頭構成（実際の順序）:

```
2. 「前提・読者層」（箇条書き5点）
3. 「著者プロフィール」（文章+箇条書き）

...（以降は技術解説が続く）
```

### 欠落点の一覧（優先度順）

**Critical（必須・今すぐ追加）**:

1. **目次が存在しない**: 冒頭にサイドバー目次も本文内目次もない。Brainの商品ページでは左サイドバーに自動生成目次が出るが、これはH2見出しを使っている場合のみ。現在の記事の見出しにはH2が使われているが、冒頭に「目次」節として番号付きリストが出ていない。競合8商品全てに目次あり。

2. **ペインポイントが冒頭にない**: 現在の冒頭は「この記事で手に入るもの（ベネフィット）」から始まっているが、売れてる商品は全て「悩み・ペイン」から始まる。「Claude Designを使ったけど日本語が崩れた」「日本語LPが作れなくて諦めた」等のペイン列挙が冒頭に必要。

3. **著者の実績数字がない**: 「文系大学生がAIだけで2ヶ月で月14万円」という実績があるのに、著者プロフィール節の書き方が謙虚すぎる。売れてる商品は「月収○万円」「○部突破」「ランキング○位」を冒頭で提示する。

4. **販売部数・[UPDATE]ログがない**: しゅうへいが発売初日から「2時間で200部突破」と更新しているように、発売後のモメンタムを示す更新ログが必要。

5. **購入者の声セクションがない**: しゅうへいは目次の第1項を「購入者の口コミ」にしている。現記事には購入者の声が全くない（発売前なので仕方ないが、モニター声などで代替が必要）。

**High（早急に改善）**:

6. **「やらないこと」「おすすめしない人」リストがない**: 対象読者の属性選別がない。トシノ・まな・こうめい全員が「こんな人にはおすすめしません」を明示している。

7. **有料ゾーン直前のフック文が弱い**: 現在確認した限りでは有料ゾーン直前の「ここから先でやること一覧」が存在しない。購入をためらっている人への最後の後押しがない。

8. **同梱物の量が見えにくい**: ベネフィットリストに8点あるが「○テンプレート」「○プロンプト」という数量表現が少ない。ユニコが「スタイルテンプレート30種類」を一個ずつ列挙しているように、量で価値を見せる必要がある。

**Medium（余裕があれば）**:

9. **競合商品との比較表の位置**: 現在は「他のAIデザインツールとの違い」という比較表が本文中盤に配置されているが、これはLP部分（無料ゾーン前半）に引き上げた方が購入動機になる。

10. **価格の正当化**: 現価格（未設定？）に対して「○本のプロンプトを一から開発すると○時間かかる」「CSSを専門家に依頼すると○円かかる」等の価格正当化フレームがない。
