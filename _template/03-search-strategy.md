# Phase 3: 検索戦略

## 目的

Phase 1-2で定義した質問リストに基づき、効果的な検索クエリを生成・実行し、情報を収集する。

## 検索クエリ生成ルール

### クエリバリエーション

各質問について **最低2〜3個のクエリバリエーション** を生成する：

- **言い換え**: 同じ内容を別の表現で検索する
- **具体化**: 固有名詞・数字・年号を含むクエリ
- **拡張**: 関連キーワードを含む広いクエリ

```
例: 「米国のAI規制の動向は？」
→ "US AI regulation 2026"
→ "US AI executive order latest"
→ "American AI policy update March 2026"
```

### ドメインprompt.mdの固定クエリ

ドメインprompt.mdの `## 検索クエリ` に記載されたクエリは **毎回必ず実行する**。
ペルソナ由来のクエリはそれに追加して実行する。

## 多言語検索ルール

| 条件 | 検索言語 |
|------|---------|
| 英語圏の国・グローバルトピック | 英語のみ |
| 非英語圏の国 | 英語 + 現地語 |
| 日本市場に特化 | 日本語メイン + 英語補助 |

- ドメインprompt.mdに言語指定がある場合はそちらを優先する
- 現地語で有益な情報が見つかった場合、記録は日本語に翻訳する

### 言語別検索テンプレート

```
英語: "[topic] [country/region] [year]"
日本語: "[トピック] [年] [キーワード]"
現地語: "[現地語トピック] [現地語キーワード] [年]"
```

## 情報ソース優先度

| 優先度 | ソースの種類 | 例 |
|--------|-----------|---|
| 最高 | 政府機関・学術機関・業界団体 | 総務省、NIST、IEEE、JEMA |
| 高 | 大手メディア・調査会社 | Reuters、Bloomberg、GfK、富士経済 |
| 中 | 専門メディア・業界レポート | TechCrunch、The Verge、ITmedia |
| 低 | 個人ブログ・SNS | Medium、X/Twitter、Reddit |

- 低優先度ソースでも、具体的な実例やデータを含む場合は有用
- ソースの種類はレポートに明記すること

## 鮮度重み付け（Freshness Weighting）

ドメインprompt.mdに鮮度テーブルが定義されている場合はそちらを使用する。
未定義の場合は以下のデフォルトを適用する：

| 鮮度 | 優先度 | 扱い |
|------|--------|------|
| 1週間以内 | 最優先 | レポートの中心情報 |
| 2週間以内 | 高 | 主要な情報として扱う |
| 1ヶ月以内 | 標準 | 通常通り記載 |
| 3ヶ月以内 | 低 | 背景情報として使用。日付を明記 |
| 3ヶ月超 | 最低 | 原則使用しない。使う場合は「⚠️ 古い情報」と明記 |

## 適応的クエリ拡張（gpt-researcher方式）

初回検索の結果を見て、**追加クエリを動的に生成する**：

1. **情報不足の軸**: 初回検索でヒットが少なかったサブ質問について、クエリを言い換えて再検索
2. **新発見の深掘り**: 検索中に見つかった重要キーワードで追加検索
3. **反証検索**: 見つかった主要な主張に対して、反対意見・批判を検索する

```
適応拡張の判断基準:
- ヒット0件 → クエリを大幅に変えて再検索（必須）
- ヒット1-2件 → クエリを微調整して追加検索（推奨）
- ヒット3件以上 → 十分。次の質問に進む
- 重要な新情報発見 → 関連クエリを追加（推奨）
```

## 検索の終了条件

- ドメインprompt.mdの全固定クエリを実行済み
- ペルソナ由来の追加クエリを実行済み
- 適応拡張を1ラウンド実施済み
- 各サブ質問に対して最低2件の情報を収集済み（0件の場合は「該当なし」と記録）
