# Phase 7: レポート生成

## 目的

Phase 6の分析結果を、読みやすいHTMLレポートとして出力する。

## HTML出力ルール

### 基本仕様
- **単一HTMLファイル**として出力する（CSS・JSはすべてインライン埋め込み）
- 外部ファイルへの依存なし（オフラインで開いても完全に表示できること）
- **文字コード**: UTF-8

### デザイン仕様

#### デフォルトテーマ: OLED Dark + Glassmorphism

```css
/* カラーパレット */
--bg-primary: #0a0a0a;
--bg-secondary: #111111;
--bg-glass: rgba(255, 255, 255, 0.05);
--border-glass: rgba(255, 255, 255, 0.1);
--text-primary: #e0e0e0;
--text-secondary: #888888;
--accent: #6366f1;       /* インディゴ */
--accent-green: #22c55e;
--accent-red: #ef4444;
--accent-yellow: #eab308;
```

#### レイアウト
- フルワイド背景 + **max-width: 1280px** コンテンツ
- 固定ナビバー（目次兼用）
- ヒーローセクション（タイトル・日付・サマリー）
- セクション間は背景色の交互切り替えで区切る
- 適切な余白・行間（**line-height: 1.7以上**）

#### インタラクティブ要素
- 目次はページ上部にアンカーリンク付きで設置
- リンクはクリック可能（`target="_blank"`）
- テーブルはホバーで行ハイライト
- チャート・グラフはCSS/SVGで実装（外部ライブラリ不使用）

#### レスポンシブ対応
- モバイル表示に対応（テーブルは横スクロール）
- `@media print` で印刷時にリンクURLが表示されるようにする

## テーマ選択

ドメインprompt.mdで `## テーマ` が指定されている場合はそちらを使用する。
未指定の場合はデフォルトテーマ（OLED Dark + Glassmorphism）を適用する。

### プリセットテーマ

| テーマ名 | 背景 | アクセント | 用途 |
|---------|------|----------|------|
| **oled-dark**（デフォルト） | #0a0a0a | #6366f1 | 汎用レポート |
| **corporate-light** | #ffffff | #1e40af | ビジネス向け・印刷想定 |
| **warm-dark** | #1a1a2e | #e94560 | 投資・金融系 |
| **nature-light** | #f8fdf5 | #16a34a | 環境・農業系 |
| **high-contrast** | #000000 | #ffffff | アクセシビリティ最優先 |

### カスタムテーマの指定方法

ドメインprompt.mdに以下を追加する:

```markdown
## テーマ
- ベース: oled-dark
- アクセント: #22c55e（緑系に変更）
- フォント: "Noto Sans JP", sans-serif
```

### theme-factoryスキルの活用

より高度なテーマが必要な場合、theme-factoryスキルを呼び出す:
- 10種のプリセットテーマ（色・フォント完備）
- 既存レポートへの後付けテーマ適用
- ドメインに合わせたカスタムテーマ生成

### ドメインprompt.mdとの連携

- レポートのセクション構成は、ドメインprompt.mdの `## レポートセクション` に従う
- セクション構成が未定義の場合、以下のデフォルト構成を使用する

### デフォルトセクション構成

1. **📊 エグゼクティブサマリー** — 全体の要約。結論から書く。3-5行
2. **🔍 詳細分析** — カテゴリ別/国別の詳細情報
3. **📈 トレンド分析** — 前回比・変化点・方向性
4. **💡 アクションアイデア** — 読者が取るべきアクション
5. **⚠️ リスク・注意点** — リスク要因・不確実性
6. **🔮 今後の注目ポイント** — 1-2週間の展望・予測
7. **📚 参考ソース** — 主要ソースのリスト

## 文章品質ルール

### 禁止パターン（使ったら即書き直し）

- **冒頭定型表現**: 「急速に進化する○○において」「目まぐるしく変化する○○」
- **ハイプ表現**: 「ゲームチェンジャー」「革命的」「画期的」「パラダイムシフト」
- **フィラー**: 「さらに」「加えて」の連続使用、「注目すべきは」の乱用
- **根拠なき主張**: ソースなしで「〜と言われている」「〜の見方が広がっている」

### 文体ルール

- **事実を先に、分析を後に。結論から書く**
- 具体的な数字・固有名詞を使う。「多くの企業が」ではなく「Google, Meta, Amazonが」
- 1段落1トピック。詰め込まない
- 読者は「で、結局どうなの？」を知りたい。**So What?を常に意識する**
- レポートの長さに上限なし。**正確さ・網羅性を優先する**（読みやすさは維持すること）

### 構造ルール（article-writing方式）

- **逆ピラミッド構造**: 最も重要な情報を冒頭に。詳細は後半に
- **1段落 = 1トピック = 3-5文**: 短く刻む。壁のような長文ブロックは禁止
- **見出しだけで内容が伝わる**: 見出しだけ読んでもレポートの骨子がわかるようにする
- **箇条書き vs 文章の使い分け**:
  - ファクトの列挙 → 箇条書き
  - 因果関係・分析 → 文章
  - 比較 → テーブル
- **数字の書き方**: 1万未満は数字（例: 5,000）、1万以上は漢字混在OK（例: 3.5兆円）
- **引用の挿入**: 重要な発言・データは引用ブロック（`<blockquote>`）で強調する

### ソースの見せ方

- インラインリンク: 文中に自然に組み込む（例: 「OpenAIが[GPT-5を発表](URL)した」）
- 末尾参照: 段落の最後にまとめて記載（例: 「（出典: [記事名](URL)）」）
- **どちらでもよいが、1レポート内では統一する**

## ソース記載ルール

- **レポート内のすべての事実情報にソースURLを付記すること（例外なし）**
- 形式: 情報の末尾に `より [タイトル](URL)` と記載する
- 1つの情報に複数ソースがある場合は全て記載する
- ソースのない情報はレポートに含めない
- 推測や一般論は「（筆者分析）」と明記する
- ソースの鮮度: 03-search-strategy.mdの鮮度ルールに従う

## デザインシステム生成（ui-ux-pro-max連携）

レポートのHTML生成前に、ui-ux-pro-maxスキルでデザインシステムを生成できる。

### 使用方法

```bash
# レポートのデザインシステムを生成
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "research report dashboard dark" --design-system -p "Search Report"

# チャートの推奨を取得
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "comparison ranking trend" --domain chart

# アクセシビリティチェック
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "accessibility contrast" --domain ux
```

### チャート選定ガイド（データ→最適なチャートタイプ）

| データの性質 | 推奨チャート | CSS/SVG実装 |
|------------|-----------|------------|
| 順位・ランキング | 横棒グラフ | `width: %` + flexbox |
| 時系列推移 | 折れ線グラフ | SVG `<polyline>` |
| 構成比 | 円グラフ / ドーナツ | SVG `<circle>` + `stroke-dasharray` |
| 比較（2-5項目） | レーダーチャート | SVG `<polygon>` |
| 分布・相関 | 散布図 | SVG `<circle>` 配置 |
| Yes/No・進捗 | プログレスバー | `width: %` + gradient |
| 国別データ | ヒートマップ風テーブル | `background-color` の濃淡 |

⚠️ チャートは外部ライブラリ不使用。CSS/SVGのみで実装すること。

### アクセシビリティ必須ルール

- テキストのコントラスト比: 最低 **4.5:1**（WCAG AA）
- テーブルの行ホバー: 視認性向上のためハイライト必須
- リンク: 色だけでなく下線でも区別可能にする
- チャート: データテーブルを併記する（スクリーンリーダー対応）
- フォントサイズ: 本文最低 **16px**、注釈最低 **14px**
