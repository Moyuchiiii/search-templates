# Phase 1: 要件明確化・サブ質問分解

## 目的

ドメインprompt.mdの調査目的を、検索可能な具体的サブ質問に分解する。

## 手順

### Step 1: 調査目的の確認

ドメインprompt.mdの `## 基本情報` から調査目的を読み取り、以下を整理する：

- **何を知りたいか**（What）: 調査の核心テーマ
- **なぜ知りたいか**（Why）: 調査結果をどう使うか
- **いつの情報か**（When）: 対象期間（直近1週間、3ヶ月以内 等）
- **どこの情報か**（Where）: 対象地域・マーケット
- **誰に向けたレポートか**（Who）: 読者のレベルと期待

### Step 2: サブ質問への分解（Decomposition）

調査目的を **3〜7個のサブ質問** に分解する。各サブ質問は：

- **1つの明確な問い**になっていること（Yes/Noで答えられない、オープンクエスチョン）
- **検索可能**であること（抽象的すぎない）
- **相互に重複しない**こと（MECE原則）
- **優先度**をつけること（必須 / あれば良い）

#### 分解テンプレート

```
Q1: [必須] ○○の現状はどうなっているか？（ファクト収集）
Q2: [必須] なぜ○○が起きているのか？（原因分析）
Q3: [必須] 今後○○はどうなるか？（予測・展望）
Q4: [あれば良い] ○○は△△とどう比較されるか？（比較分析）
Q5: [あれば良い] ○○から得られるアクションは何か？（実用的示唆）
```

### Step 3: スコープ定義

| 項目 | 定義 |
|------|------|
| 対象期間 | ドメインprompt.mdの指定に従う |
| 対象地域 | country.md参照 or ドメイン指定 |
| 情報の深さ | Tier分けがある場合はTier別に粒度を変える |
| 除外条件 | 調査対象外とするトピック・ソースがあれば明記 |

### Step 4: 出力

分解結果を以下のフォーマットで調査結果ファイルの冒頭に記録する：

```
## 調査設計

### 調査目的
（1-2文で要約）

### サブ質問
1. [必須] ○○○？
2. [必須] △△△？
3. [あれば良い] □□□？

### スコープ
- 対象期間: YYYY-MM-DD 〜 YYYY-MM-DD
- 対象地域: ○○
- 除外: △△
```

## 適応ルール

- ドメインprompt.mdに `## 調査カテゴリ` が明示されている場合、そのカテゴリをサブ質問の軸として採用する（自動分解は補助的に使う）
- 繰り返し実行される調査（週次等）では、前回レポートとの差分を意識したサブ質問を追加する
