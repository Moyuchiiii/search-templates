# Phase 6: 分析・スコアリング

## 目的

Phase 4-5で検証済みの情報を構造化し、スコアリング・ランキング・トレンド分析を行う。

## 手順

### Step 1: 情報圧縮（open_deep_research方式）

大量の生データをレポートに変換する前に、情報を圧縮・構造化する。

#### 圧縮ルール
- 同じトピックについて複数ソースがある場合、最も信頼性の高いソースを主軸にする
- 重複情報は統合し、追加ソースは補強として引用する
- 数値データは最新の出典を優先する
- 圧縮後も**ソースURLは必ず保持する**（トレーサビリティ確保）

#### 圧縮フォーマット

```
### [トピック名]

**要点**: [1-2文で核心を記述]
**詳細**: [補足情報。3-5文]
**数値**: [具体的な数字があれば]
**ソース**: [URL1], [URL2]
**信頼度**: [検証済/未検証] / コンセンサス[高/中/低]
**鮮度**: [情報の日付]
```

### Step 2: スコアリング（ドメインprompt.md連動）

ドメインprompt.mdに `## スコアリング設定` がある場合、その軸と重みに従ってスコアリングする。

#### 汎用スコアリングフレームワーク

スコアリング設定がない場合でも、以下の汎用フレームワークを適用できる：

```
総合スコア = Σ (各軸のスコア × 重み)

各軸のスコア: 1〜10の整数
重み: 0〜1（合計1.0）
信頼度補正: スコアに検証状態の係数を掛ける
  - 検証済み: ×1.0
  - 未検証: ×0.7
  - ソース不足: ×0.5
```

#### スコアリングテーブルのフォーマット

| 順位 | 対象 | 軸1 | 軸2 | ... | 総合スコア | 信頼度 |
|------|------|-----|-----|-----|----------|--------|
| 1 | ○○ | 8 | 7 | ... | 85 | ★★★★☆ |

### Step 3: トレンド分析

前回レポートが存在する場合、以下の差分分析を行う：

- **新規**: 今回初めて登場した情報・トレンド
- **変化**: 前回から状態が変わった情報（`[変化]`タグ）
- **継続**: 前回から変化なしの情報
- **消失**: 前回はあったが今回確認できなくなった情報

### Step 4: So What? テスト

各分析結果に対して「で、結局どうなの？」テストを行う：

- この情報は読者にとってなぜ重要か？
- 読者は何をすべきか？（アクション）
- 何も思いつかない場合、その情報はレポートから除外を検討する

### Step 5: 市場分析フレームワーク（market-research方式）

ドメインprompt.mdが市場調査・競合分析を含む場合、以下のフレームワークを適用する。

#### 市場規模の推定（必須: トップダウン + ボトムアップ）

どちらか一方だけでは不十分。両方で検証する:

- **トップダウン**: 業界レポート・統計データから全体市場→対象セグメントに絞り込む
  - 例: 「AI市場$500B → そのうちAI SaaS市場$80B → 中小企業向け$15B」
- **ボトムアップ**: 顧客数×単価×頻度で積み上げる
  - 例: 「想定顧客1,000社×月額$200×12ヶ月 = $2.4M/年」

#### 競合分析テンプレート

| 項目 | 自社/対象 | 競合A | 競合B |
|------|---------|-------|-------|
| 価格帯 | | | |
| 主要機能 | | | |
| ターゲット | | | |
| 強み | | | |
| 弱み | | | |
| 差別化ポイント | | | |

#### 意思決定マトリクス

スコアリングだけでなく、最終的な推奨には以下の視点を加える:

- **リスク調整後リターン**: 高スコアでもリスクが高ければ推奨度を下げる
- **実行可能性**: 技術・リソース・時間の制約を考慮
- **タイミング**: 今やるべきか、待つべきか（市場の成熟度）

## 信頼度スコア（情報充実度）

各カテゴリ/国の情報充実度を5段階で評価する：

| スコア | 基準 |
|--------|------|
| ★★★★★ | 複数の独立ソースで十分な情報。検証済み多数 |
| ★★★★☆ | 主要な動向は把握できている。一部未検証あり |
| ★★★☆☆ | 基本情報は取得できたが網羅性に不安 |
| ★★☆☆☆ | 情報が限定的。記載内容の精度に注意 |
| ★☆☆☆☆ | ほぼ情報なし。「⚠️ 情報不足」と明記 |
