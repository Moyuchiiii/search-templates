# Phase 4: データ収集・記録

## 目的

検索で得られた情報を構造化して記録する。このファイルで定義するフォーマットは、Phase 5（検証）・Phase 7（レポート）の入力データとなる。

## 出力ファイル

`調査結果_YYYY-MM-DD.md` に以下のフォーマットで記録する。

## フォーマット

### ファイル冒頭

```markdown
# [調査名] 調査結果 YYYY-MM-DD

## 調査設計
→ Phase 1 の出力は prompt.md および 01-clarify の出力を参照

## ペルソナ
→ Phase 2 の出力は prompt.md の「ペルソナ設定」を参照

## 調査ログ（全クエリ一覧）

| # | カテゴリ/国 | 検索クエリ | 言語 | ヒット数 | 採用数 | 適応拡張 |
|---|-----------|-----------|------|---------|--------|---------|
| 1 | アメリカ | "US AI policy March 2026" | EN | 5件 | 3件 | — |
| 2 | アメリカ | "OpenAI GPT-5 2026" | EN | 8件 | 2件 | — |
| 3 | アメリカ | "AI chip export control latest" | EN | 3件 | 2件 | 適応拡張 |

（使用した検索クエリをすべて記録する。省略禁止。）
```

### カテゴリ/国 別セクション

```markdown
## [カテゴリ名 or 国名] [国旗]

### 使用した検索クエリ
1. "クエリ1" → N件ヒット、N件採用
2. "クエリ2" → N件ヒット、N件採用
3. [適応拡張] "クエリ3" → N件ヒット、N件採用

### 収集した情報

- **[記事タイトル/トピック]**
  - ソース: [記事タイトル](URL)
  - 取得日: YYYY-MM-DD
  - 鮮度: [24時間以内 / 3日以内 / 1週間以内 / 2週間以内 / 1ヶ月以内 / それ以上]
  - 信頼度: ✅ 一次データ / 🔶 二次データ / ⚠️ 参考情報
  - 検証状態: [未検証] / [検証済]（Phase 5で更新）
  - 要点（3〜5行）: この記事の核心的な主張・発見を簡潔にまとめる（「So What?」に答える内容。事実の羅列ではなく意味を書く）
  - キーファクト:
    - [具体的な数字]（例: 「市場規模$7.63B → $182.97B（2033年予測、出所: Grand View Research）」）
    - [固有名詞]（例: 「Google, Meta, Amazonが参入」）
    - [重要な引用]（例: 「CEOは"2026年末までに全社導入"と発言」）
  - 信頼度タグ: [事実] / [分析] / [推論] / [未検証]
  - 全文: `raw/[連番]_[記事タイトルの短縮].txt` に保存済み

### ソース不明の情報（信頼性低）

- **[トピック]**
  - ソース: なし
  - 信頼度: ⚠️ ソースなし（信頼に欠ける情報）
  - 要点: ～～～
  - 備考: SNSで言及を確認したがソース元記事が見つからなかった 等

### 不採用にした情報

- [記事タイトル](URL) — 不採用理由: 古い情報（2024年の記事）
- [トピック](ソースなし) — 不採用理由: ソース不明

### 信頼度サマリー

- 情報充実度: ★☆☆☆☆〜★★★★★
- ソースの質: 一次データN件、二次データN件、参考情報N件
- 鮮度: [主要な情報の鮮度帯]
```

## 全文保存（rawフォルダ）

調査結果MDと同階層に `raw/` フォルダを作成し、収集した記事の全文をソース別テキストファイルで保存する。

```
[ドメイン名]/YYYY-MM-DD/
  調査結果_YYYY-MM-DD.md     ← 要点+キーファクト記録
  raw/
    01_記事タイトルの短縮.txt
    02_記事タイトルの短縮.txt
    ...
```

- Phase 5-6 で詳細が必要になった場合のみ、rawフォルダから該当ファイルを読み込む
- 調査結果MDには全文を貼り付けない。要点+キーファクトのみを記載する

## 記録ルール（厳守）

### 絶対ルール
- 検索クエリは使ったもの **全件** を記録する。1つも省略しない
- **キーファクト（数字・固有名詞・引用）は必ず残す。これらの省略は禁止**
- 「特になし」と記載する場合も、どのクエリで検索して該当がなかったかを記録する
- **すべての情報にソースURL（出典元リンク）を付記する**
- ソースが確認できない情報は「⚠️ ソースなし」として **分離記録する**

### 数値データの記録
- 市場規模・投資額・ユーザー数等の数値には、**必ず出典年と出所を記載する**
- 例: 「500億ドル（IDC, 2026年3月推計）」
- 通貨は原文のまま記録し、必要に応じて円換算を併記する
- 1ソースあたりキーファクト **3個以上** を目安とする（定量情報がない場合はその旨を明記）

### 情報の分類タグ
- `[事実]` — 確認された事実（ソースあり）
- `[分析]` — ソースの著者による分析・見解
- `[推論]` — 調査者（AI）による推論。根拠を明記
- `[未検証]` — 単一ソースのみで確認。Phase 5で検証予定
- `[検証済]` — 2つ以上の独立ソースで確認済み
